# アーキテクチャ

## 設計思想

### 1. ミラー原則

ディレクトリ構造は `~/.claude/` のレイアウトをミラーします。開発者は Claude Code のファイル配置を知っていれば、このリポジトリのどこに何があるかを理解できます。

### 2. カテゴリ優先

Skills と Agents はファイル種別ではなく問題ドメインで分類します（例: `git/`, `review/`）。これにより関連ツールをまとめて見つけられます。

### 3. 自己完結ユニット

各 Skill・Agent は独自の `README.md` を持ち、外部ドキュメントなしに理解できます。

### 4. 明示的なインストール

何も自動実行しません。インストールは常に明示的・冪等です。

## インストール先の対応

| このリポジトリ                         | インストール先（グローバル） |
| -------------------------------------- | ---------------------------- |
| `skills/<category>/<name>/`            | `~/.claude/skills/<name>/`   |
| `agents/<category>/<name>/`            | `~/.claude/agents/<name>/`   |
| `claude-md-templates/<type>/CLAUDE.md` | `<project>/CLAUDE.md`        |

Skills と Agents はカテゴリサブディレクトリが挟まっていますが、インストール先はフラットになります。
これはリポジトリでの閲覧性と Claude Code の要求する構造を両立するためです。

## グローバル vs プロジェクト

- **グローバル（シンボリックリンク推奨）**: リポジトリ更新が即座に全プロジェクトに反映される。個人利用に最適。
- **プロジェクト（コピー推奨）**: プロジェクトが自己完結する。チーム共有・再現性が重要な場合に最適。

## ファイル種別の役割

| ファイル             | 読み手              | 目的                                   |
| -------------------- | ------------------- | -------------------------------------- |
| `SKILL.md`           | Claude Code（機械） | スラッシュコマンドの定義・動作         |
| `<name>.md`（Agent） | Claude Code（機械） | エージェントのシステムプロンプト・設定 |
| `README.md`          | 人間                | 使い方・例・インストール手順           |
| `docs/*.md`          | 人間                | 設計思想・オーサリングガイド           |
