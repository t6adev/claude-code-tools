# ドキュメントテンプレート

`analyze-project` スキルが生成するドキュメントのテンプレート。
探索結果をこのテンプレートに埋め込んでファイルを生成する。

---

## CLAUDE.md

```markdown
# [プロジェクト名]

## 概要

[プロジェクトの目的・解決する課題（2〜3文）]

## 技術スタック

- Frontend: [例: Next.js 15 (App Router) + TypeScript]
- Backend: [例: Next.js API Routes / Hono]
- Database: [例: PostgreSQL (Supabase)]
- ORM: [例: Prisma 5]
- 認証: [例: NextAuth.js v5]
- インフラ: [例: Vercel]
- CSS: [例: Tailwind CSS v4]
- テスト: [例: Vitest + Playwright]

## 開発規約

- パッケージマネージャー: [pnpm / npm / yarn / bun]
- コードスタイル: [例: oxlint + Prettier]
- コミット規約: [例: Conventional Commits]
- ブランチ戦略: [例: GitHub Flow / Gitflow]

## よく使うコマンド

- 開発サーバー起動: `[コマンド]`
- テスト実行: `[コマンド]`
- リント・型チェック: `[コマンド]`
- ビルド: `[コマンド]`
- DBマイグレーション: `[コマンド]`（該当する場合）

## ディレクトリ構成

[実際の構成を反映したテキスト図]

## 実装規約・パターン

[コードから読み取った慣行（命名規則・ファイル配置・状態管理など）]

## 注意事項・禁止事項

[既存コードから推測できる設計上の制約・アンチパターン]
```

---

## ドキュメント出力のルール

- コードから読み取れなかった項目は `[要確認]` とマークして空欄にしない
- 推測した内容には `（コードから推測）` と注記する
- 生成後にユーザーへ「以下の項目は確認・補足が必要です」と未確定箇所を列挙する
